{% include tr.html s="how to install landscapes" %}

{% include tr.html s="After you have downloaded the .zip file for a landscape from this page, you need to install it in Stellarium." %}

{% include tr.html s="automatic" %}

{% include tr.html s="If you have Stellarium 0.10.6 or later version, you can use the 'Add/remove landscapes' feature to install landscapes automatically:" %}

  • {% include tr.html s="Open the 'Sky and viewing options' window by clicking on the appropriate button in the left button bar (or press the F4 key)." %}
  • {% include tr.html s="The 'Add/remove landscapes' button is at the bottom of the 'Landscape' tab." %}
  • {% include tr.html s="When you press it, the 'Add/remove landscapes' window will appear. It allows you to install .zip files containing landscapes. It also lists the user-installed landscapes and allows you to remove them." %}

{% include tr.html s="Note that while this makes installing landscapes easier, it may also cause you to overlook what else is included in the ZIP archive." %} {% include tr.html s="Landscape packages created without this feature in mind may contain other files, such as alternative textures in different sizes." %}

{% include tr.html s="manual" %}

{% include tr.html s="If you are using an earlier version of Stellarium, you can follow this procedure to install a landscape package:" %}

  • {% include tr.html s="Browse to your User Data Directory, which varies according to your operating system." %}
  • {% include tr.html s="Create a sub-directory called landscapes in your user directory (if it doesn't exist)." %}
  • {% include tr.html s="Unzip the landscape .zip file in the landscapes directory (if it's done right, a sub-directory should be created for each landscape)." %}

Please include a location section in your landscape.ini file with the longitude, latitude, altitude and planet for the location of the landscape (see one of the pre-existing landscapes for an example)." %}

{% include tr.html s="To find out more about how to create a landscape, see the Stellarium User Guide, and examine existing landscapes. If you are having problems, posting to the forums is a good way to get some advice." %}

{% include tr.html s="important note on image dimensions" %}

{% include tr.html s="IMPORTANT: Make sure all textures have dimensions which are integer powers of 2, i.e. 256, 512, 1024, 2048, 4096, 8192, 16384, ... e.g. 4096 by 1024, 2048 by 2048 and so on." %}

{% include tr.html s="This is a limitation of OpenGL. Some video hardware will work OK with images with different image dimensions, but many will not display properly, suffer vastly reduced frame rates, and even crash the computer." %}

{% include tr.html s="Please make sure all contributed landscapes conform to these requirements, or your link may be removed." %}

{% include tr.html s="Be aware that many people's video hardware cannot handle very large textures. This is hardware and driver dependent. A typical maximum image size is 2048x2048 or 4096x4096." %}

{% include tr.html s="package contents" %}

{% include tr.html s="Please package your landscape in a .zip file with all files inside a directory in the .zip file. This should be unique to your landscape, and it would be nice if it was all lower-case with no spaces." %}

{% include tr.html s="You should also include a readme.txt file which describes the landscape and specifies any usage restrictions or licensing terms for the images used in the landscape." %}

{% include tr.html s="licensing" %}

{% include tr.html s="Before you distribute images as part of a Stellarium landscape, please ensure you are legally entitled to - you must be the copyright holder for the images, or be able to distribute them for use with Stellarium under the terms of some agreement with the copyright holder (e.g. Creative Commons licensed images found on the web)." %}

{% include tr.html s="It is important to explicitly state what use may be made of images for your landscape. This should be done in the readme.txt file inside the .zip file." %}

{% include tr.html s="We recommend an open source license compatible with Stellarium itself (i.e. the GNU GPL), or one of the Creative Commons licenses." %}

{% include tr.html s="file encoding" %}

{% include tr.html s="The landscape.ini and readme.txt files should be UTF-8 encoded text or plain ASCII. It's probably a good idea to adopt the Windows line ending encoding, (i.e. CR LF). Both Windows and *nix style line ending encoding should work OK in Stellarium, but Windows users will have an ugly time reading the readme.txt if it uses *nix-style newlines." %}
