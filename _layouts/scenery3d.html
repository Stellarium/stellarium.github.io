{% include tr.html s="how to install 3D sceneries" %}

{% include tr.html s="After you have downloaded the .zip file for a 3D scenery, you need to install it in Stellarium." %}

  • {% include tr.html s="Browse to your User Data Directory, which varies according to your operating system. (See User Guide)" %}
  • {% include tr.html s="Create a sub-directory called scenery3d in your User Data Directory (if it doesn't exist)." %}
  • {% include tr.html s="Unzip the scenery .zip file in the scenery3d directory (if it's done right, a sub-directory should be created for each scenery)." %}
  • {% include tr.html s="If there is also a landscapes subdirectory in the ZIP file, move its contents to the landscapes subdirectory." %}

{% include tr.html s="important note on image dimensions" %}

{% include tr.html s="IMPORTANT: Try to still use textures with dimensions of integer powers of 2, i.e. 256, 512, 1024, 2048, 4096, 8192, 16384, ... e.g. 4096 by 1024, 2048 by 2048 and so on." %}

{% include tr.html s="This is a limitation of older OpenGL hardware. Most video hardware will nowadays work OK with images with different image dimensions, but some will not display properly, suffer vastly reduced frame rates, and even crash the computer." %}

{% include tr.html s="Be aware that many people's video hardware cannot handle very large textures. This is hardware and driver dependent. Maximum compatibility is achieved with image size not exceeding 2048x2048." %}

{% include tr.html s="package contents" %}

{% include tr.html s="Please package your 3D scenery in a .zip file with all files inside a subdirectory in the .zip file. This should be unique to your scenery, and must be all lower-case with no spaces." %}

{% include tr.html s="If you enclose your scenery with a landscape, provide this in a landscapes/name subdirectory in the ZIP file." %}

{% include tr.html s="The file description.en.utf8 (and any translated versions which you may want to provide yourself) describes the scenery and what is special about it. Please make this a self-contained explanation that does not only point to external links." %}

{% include tr.html s="You should also include a readme.txt file which briefly describes the landscape and specifies any usage restrictions or licensing terms for the images used in the 3D scenery." %}

{% include tr.html s="licensing" %}

{% include tr.html s="Before you distribute images as part of a Stellarium 3D scenery or landscape, please ensure you are legally entitled to - you must be the copyright holder for the images, or be able to distribute them for use with Stellarium under the terms of some agreement with the copyright holder (e.g. Creative Commons licensed images found on the web)." %}

{% include tr.html s="It is important to explicitly state what use may be made of images for your 3D scenery. This should be done in the readme.txt file inside the .zip file." %}

{% include tr.html s="We recommend an open source license compatible with Stellarium itself (i.e., the GNU GPL), or one of the Creative Commons licenses." %}

{% include tr.html s="file encoding" %}

{% include tr.html s="The scenery3d.ini and readme.txt files should be UTF-8 encoded text or plain ASCII. It's probably a good idea to adopt the Windows line ending encoding, (i.e. CR LF). Both Windows and *nix style line ending encoding should work OK in Stellarium, but Windows users will have an ugly time reading the readme.txt if it uses *nix-style newlines." %}
